mini2440开发环境搭建总结

发表于 2016-04-04   |   分类于 开发环境与工具   |  

文章系原创，如需转载请注明：转载自”Blog of UnicornX” (http://unicornx.github.io/)

最新更新于：2016-04-04

通过supervivi更新u-boot

前提：确保Norflash中存在supervivi

  • 启动选择Norflash方式
  • 上电进入supervivi界面
  • 菜单选择功能号[v]，进入等待下载模式
  • 打开DNW 程序，接上USB 电缆，如果DNW 标题栏提示[USB：OK]，说明USB 连接成功
  • 点击DNW的“USB Port->Transmit/Restore”选项，并选择打开文件开始下载。
  • 关机
  • 改变启动选择Nandflash方式
  • 上电进入u-boot界面

基于mini2440搭建Linux开发调试环境

开发方式基于u-boot，在u-boot启动过程中采用tftp下载内核镜像(注意因为是采用uboot下载引导，所以这里的内核应该是uImage才对，否则引导时报格式错误)，然后使用nfs挂载根文件系统进行调试。当然也可以全部采用nfs将两者全部搞定。但目前暂不考虑。

从调试的角度出发，我们需要保证，host的有线网卡，client(即虚拟机Ubuntu的eth1)以及mini2440开发板的ip地址在同一个网段内，这样三者之间才能互通。

在这里假设:

  • host的无线网段为dhcp方式，获取的ip地址为192.168.1.102，host的有线网卡设置静态方式为192.168.0.100
  • client的eth0设置为192.168.1.110，client的eth1设置为192.168.0.110
  • dev-board的ip地址设置为192.168.0.2

host(ubuntu)上需要准备好:

  • network:

    基本物理联网方式参考嵌入式开发环境中如何设置VMware的网络 中的方式三。

    设置网口的方式大致如下：

    $ sudo vi /etc/network/interfaces

    编辑/etc/network/interfaces如下：

    #############################################
# Bridge for WLAN, used for ssh login from the host
# ip address should be in the same sub-network as wireless
# interface of the host machine
auto eth0
iface eth0 inet static

address 192.168.1.110
netmask 255.255.255.0
gateway 192.168.1.1
dns-nameservers 114.114.114.114

#############################################
# Bridge for Fixed-Line, used for connecting with
# dev-board, should be in the same sub-network as the
# dev-board
auto eth1
iface eth1 inet static

address 192.168.0.110
netmask 255.255.255.0
gateway 192.168.0.1

    保存后退出，重启网口如下：

    $ sudo service networking restart

  • tftp:

    执行如下操作:

    $ sudo apt-get install tftpd-hpa tftp-hpa
$ mkdir /home/fs/ws/mini2440/tftp
$ chmod 777 /home/fs/ws/mini2440/tftp
$ sudo vi /etc/default/tftpd-hpa

    编辑/etc/default/tftpd-hpa如下：

    TFTP_USERNAME="tftp"
TFTP_DIRECTORY="/home/fs/ws/mini2440/tftp"
TFTP_ADDRESS="0.0.0.0:69"
TFTP_OPTIONS="--secure"

    保存后退出，重启tftp服务如下：

    $ sudo service tftpd-hpa restart

  • nfs:
    执行如下操作:

    $ sudo apt-get install nfs-kernel-server
$ mkdir /home/fs/ws/mini2440/nfs/rootfs
$ sudo vi /etc/exports

    编辑/etc/exports如下：

    /home/fs/ws/mini2440/nfs/rootfs *(rw,sync,no_root_squash,no_subtree_check)

    保存后退出，重启nfs服务如下：

    $ sudo service nfs-kernel-server restart

dev-board

  • u-boot:

    设置环境变量如下：

    bootdelay=1
baudrate=115200
ethaddr=08:08:11:18:12:27
ipaddr=192.168.0.2
gatewayip=192.168.0.1
netmask=255.255.255.0
tekkaman=bmp d 70000
stdin=serial
stdout=serial
stderr=serial
ethact=dm9000
serverip=192.168.0.110
bootargs=noinitrd root=/dev/nfs rw nfsroot=192.168.0.110:/home/fs/ws/mini2440/nfs/rootfs ip=192.168.0.2:192.168.0.1::255.255.255.0 console=ttySAC0,115200 init=/linuxrc mem=64M
bootcmd=tftp 0x30008000 zImage; bootm 0x30008000

关键是其中的如下几项要设置好：
- ipaddr
- serverip
- bootargs
- bootcmd

supervivi使用功能总结

功能[a]：通过USB下载用户程序到Nand Flash的起始地址Block 0中，reset重启后，S3C2440会自动将程序加载到内部SRAM(StepStone)中运行。可以用于烧写裸机程序。如果程序不超过4K则可以自动执行，如果超过4K则需要采用搬移等操作，像U-boot那样。

功能[d]：通过USB下载程序到指定内存地址(通过DNW 的Configuration->Option 选项指定运行地址)，并运行。对于本开发板，SDRAM 的物理起始地址是0x30000000，结束地址是0x34000000，大小为64Mbytes，另外BIOS(应该指的就是supervivi)本身占用了0x33DE8000 以上的空间，因此在用supervivi的的USB下载功能时应指定地址在0x30000000~0x33DE8000之间。

Linux内核模块的Makefile写法分析

发表于 2016-01-13   |   分类于 Linux驱动开发   |  

文章系原创，如需转载请注明：转载自”Blog of UnicornX” (http://unicornx.github.io/)

以下描述针对Ubuntu环境。

# General Purpose Makefile for Linux Kernel module  
# 以下编译适合于在内核代码树之外编译内核模块。
# 整个过程会导致本Makefile文件被加载执行两遍。
# 第一遍执行时，由于`KERNELRELEASE`没有被定义，所以先执行`else`下的逻辑。
# `-C $(KERNEL_DIR)`指明跳转到内核源码目录下读取那里的Makefile；
# `M=$(CURDIR)`表明然后返回到当前目录继续读入当前的Makefile并执行之。此为第二遍执行。
# 第二遍执行时，此时从内核源码目录返回时，KERNELRELEASE已被被定义，
# kbuild也被启动去解析kbuild语法的语句，make将继续读取else之前的内容。
# else之前的内容为kbuild语法的语句, 指明模块源码中各文件的依赖关系，以及要生成的目标模块名。

ARCH        := arm        # 如果为PC机开发则赋值为空  
CROSS_COMPILE    := arm-linux-    # 如果为PC机开发则赋值为空

KERNEL_VER    := $(shell uname -r)

KERNEL_DIR    ?= /path-to-linux-kernel-source-tree    # 适用于嵌入式开发的交叉编译环境
#KERNEL_DIR    ?= /usr/src/$(KERNEL_VER)        # 适用于PC机编译环境，已下载了内核对应的源代码
#KERNEL_DIR    ?= /lib/modules/$(KERNEL_VER)/build    # 适用于PC机编译环境，无需下载内核对应源代码，利用升级内核时缺省下载的build目录

ifneq ($(KERNELRELEASE),)

    # call from kernel build system
    obj-m      := file.o

else

default:
    $(MAKE) ARCH=$(ARM) CROSS_COMPILE=$(CROSS_COMPILE) -C $(KERNEL_DIR) M=$(CURDIR) modules

endif

clean:                                     
    $(MAKE) -C $(KERNEL_DIR) M=$(CURDIR) clean

如果一个模块包括多个.c文件(如file1.c、file2.c)，则应该以如下方式编写Makefile中的obj-m：

obj-m := modulename.o
modulename-objs := file1.o file2.o

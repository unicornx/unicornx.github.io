Node.js + MongoDB 开发环境搭建

前言

Windows下环境搭建

参考

  • Win7下MongoDB的安装和部署测试
  • Easy web - node.js + mongodb (1): 环境搭建

注：这里使用的Windows系统为Windows7 64位版本。

安装Node.js

  • 进入Node.js的官网 下载for Windows的安装包，建议使用LTS的版本。我下载的是v4.4.4 LTS的版本，与之对应的最新的版本已经到v6.2.0，但如果不是为了尝试最新的功能，不建议下载最新的开发版本。

  • 下载后是一个后缀名为msi的安装包，譬如node-v4.4.4-x64.msi，这里我的Windows系统是64位。双击后直接安装。缺省安装在C:\Program Files\nodejs目录下。查看C:\Program Files\nodejs\node_modules目录下只有一个干净的npm。因为我们的开发工程已经自带了所有必须的package，所以这里并不需要再额外安装其他的packages。

  • 测试一下是否安装成功

    win + r，输入cmd后回车，打开命令窗口，在命令窗口中输入: node -v, 回车. 若安装成功, 则会显示 node 的版本号。

安装MongoDB

官方安装指导可以参考Install MongoDB Community Edition on Windows。

  • 进入MongoDB官网的下载中心。根据系统版本选择对应的MongoDB版本下载，譬如系统为Windows7 64bit，则选择Windows的64bit版本，选择”Community Server”系列就好了。

  • 下载后是一个后缀名为msi的安装包，譬如mongodb-win32-x86_64-3.2.4-signed.msi。双击后直接安装。步骤参考Win7下MongoDB的安装和部署测试一直走到第5步为止。缺省安装在C:\Program Files\MongoDB\目录下。同时

  • 我们会发现所有的可执行程序都放在C:\Program Files\MongoDB\Server\3.2\bin下。为方便后面命令的执行，将这个目录添加到系统路径中。方法参考怎样打开并设置win7系统环境变量。把上述C:\Program Files\MongoDB\Server\3.2\bin路径添加到系统变量的Path中。

  • 安装MongoDB服务

    首先确保MongoDB需要的存放数据库文件和日志文件的目录必须存在。MongoDB默认的数据库目录为：C:\data\db。但也可以在安装服务时自己指定。我们采用的是自己在命令行中自己指定。

    假设我们将数据库文件存放在D:\temp\data\db，将日志文件存放在D:\temp\ data\log，手动创建以上目录。

    然后win + r，输入cmd后回车，打开命令窗口，在命令窗口中输入:

    mongod.exe --dbpath "D:\temp\data\db" --logpath "D:\temp\data\log\1.log" --install --serviceName "mongodb"

    --serviceName这个参数是用于指定我们安装的服务的名字。可以在Windows的服务管理器里看到，参考Win7下MongoDB的安装和部署测试的步骤8

  • 启动/停止MonggoDB服务

    win + r，输入cmd后回车，打开命令窗口，在命令窗口中输入:

    • 启动服务：net start mongodb
    • 停止服务：net stop mongodb

  • 测试服务

    win + r，输入cmd后回车，打开命令窗口，在命令窗口中输入: mongo.exe，如果能看到欢迎界面即说明工作正常，譬如：

    ...
connecting to: test
Welcome to the MongoDB shell.

貌似如此这般以后就ok了，不需要参考Easy web - node.js + mongodb (1): 环境搭建的1.3.2 把mongo设置成一个windows的服务所述设置，机器重启后服务依然可用。

运行项目

win + r，输入cmd后回车，打开命令窗口。进入项目源码目录，譬如D:\ws\webrtc\WebRtc0.5，运行如下命令：

D:\ws\webrtc\WebRtc0.5>node app.js

打开浏览器，地址栏输入http://localhost:8080/，回车即可看到系统主页面。

Linux下环境搭建

参考

  • Ubuntu 14.04下搭建Node.js开发环境 里面有如何在Ubuntu中搭建WebStorm的介绍
  • Installing Node.js via package manager 官方的安装说明
  • [如何在Ubuntu14.04服务器上安装NodeJS(]http://www.nodejs.net/a/20141106/145909.html)

注：这里使用的Ubuntu系统为14.04 LTS 64位版本。

安装Node.js

  • 进入Node.js的下载官网 下载Source Code的tar包，建议使用LTS的版本。我下载的是node-v4.4.4.tar.gz。

  • 下载后执行如下操作

    先安装编译工具，否则后续操作configure时会报警WARNING: failed to autodetect C++ compiler version (CXX=g++)，具体参考编译安装nodejs不通过，请问怎么破(搞定了）

    $ sudo apt-get install build-essential

    然后按如下步骤进行编译和安装

    $ tar xzf node-v4.4.4.tar.gz 
$ cd node-v4.4.4/
$ ./configure 
$ make
$ sudo make install

  • 测试一下是否安装成功

    输入: node -v, 回车. 若安装成功, 则会显示 node 的版本号。

安装MongoDB

官方安装指导可以参考Install MongoDB Community Edition on Ubuntu。

但问题是如果从官方的repo下载速度奇慢，参考Mongodb 安装慢到极致，所以我们还是使用ubuntu原生提供的版本，虽然版本老一点但安装速度快得多。

  • 安装步骤

    $ sudo apt-get install mongodb

  • 安装完毕后缺省服务已经启动

运行项目

进入项目源码目录，譬如~/ws/WebRtc0.5，运行如下命令：

$ node ./app.js

打开FireFox浏览器，地址栏输入http://localhost:8080/，回车即可看到系统主页面。

用hexo发布文章

发表于 2015-06-28   |   分类于 我的博客   |  

文章系原创，如需转载请注明：转载自”Blog of UnicornX” (http://unicornx.github.io/)

简单地记录一下自己使用hexo写博客，发布文章的经验总结。

创建一篇新文章很容易，执行如下命令：

hexo new <layout> "filename"

其中表示文章的模版类型，对应的是你的博客项目顶层目录下的scaffolds目录里的模版名字。

filename就是你创建的文章对应的文件的名字了，创建成功后会对应生成一个markdown文件。注意filename要用双引号引起来，如果你的filename中有空格，最后对应的文件名中空格会被”-“代替。建议这里的filename中全部是英文或字母，不要出现中文。文章的标题可以在正文中的title里面再确定，那里才是会被显示到页面上的文字，虽然缺省模版里会用你这里输入的filename来填写title域，文件生成后再改也不迟。

另外一个在正式写文章之前需要注意的问题就是有关模版文件的定制。模版文件开头的两行用来表示文章的标题域(title), 创建的时间域(date) ，届时这两个域hexo会自动帮你插入内容。可以自己添加分类(categories)和标签(tags)域。我自己目前用的post和draft的模版看起来是这个样子的：

title: {{ title }}
date: {{ date }}
categories:
tags:
---

每次new一个post时，hexo会基于该模版文件创建你的blog文件，”—“下面就是你输入文字的地方了。
注意：如果你需要输入中文，转化称html后可能会是乱码，解决方法，将markdown文件另存为UTF-8格式就好了，所以为了方便起见，我们可以先把这些模版文件先另存为UTF-8格式，以后对每个new文件就不用转了。

layout 对应的模版文件名 创建好的文章存放的位置 用途
post post.md source_posts 正式发布的文章
draft draft.md source_draft 草稿
page page.md html页面

所以要正式创建一篇文章并且发布可以输入以下命令：

$ hexo new post ”file name“

或者

$ hexo new ”file name“

即缺省不给就是post。

如果你想先写一些文章，暂时不发布，那就是要用到draft(草稿)的功能了。

步骤如下：

先创建一个草稿

$ hexo new draft ”file name"

会在source/_drafts目录下生成一个文件file-name.md。但是这个文件不被显示在页面上，链接也访问不到。也就是说如果你想把某一篇文章移除显示，又不舍得删除，可以把它移动到_drafts目录之中。

本地预览的时候可以加个参数，就可以看见draft的文章了。

$ hexo server --drafts

下面这条命令可以把草稿变成文章:

$ hexo publish post "file-name"

注意这里file-name不要包含文件名的后缀部分.md。

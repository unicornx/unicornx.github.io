尝试在文本模式下在VMware上运行ubuntu

发表于 2015-07-10   |   分类于 开发环境与工具   |  

文章系原创，如需转载请注明：转载自”Blog of UnicornX” (http://unicornx.github.io/)

最新更新于：2016-03-04

一、问题：

平时工作一直在VMware的环境下运行Ubuntu，Ubuntu升级到12.04LTS后VMware缺省需要1个G的内存才可以运行，自己的本本总共物理内存才2个G，实在感觉有点捉襟见肘，但又舍不得银子，所以决定研究一下是否可以在现有基础上优化一下。

二、解决方法

关键是要降内存消耗，我尝试的方法就是采用纯文本模式运行Ubuntu，不开启图形模式，方法如下：
第一步，修改文件/etc/default/grub，注意需要root权限：

$ sudo vi /etc/default/grub

找到这一行:
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash"
添加一个选项text如下:
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash text"

第二步，更新grub

$ sudo update-grub

第三步，重启生效

$ sudo reboot

如果要切换回图形模式，只要重复以上三步，并在第一步中确保去掉text选项即可。

经测试发现，相同运行条件下，原来VMWare要为我的Ubuntu12.04LTS虚拟机配置1个G的内存，现在似乎改为512M跑起来也蛮流畅的，呵呵。

三、锦上添花

锦上添花之一，使用Putty做前端

图形模式设置好后，直接在纯文本模式下运行有个小问题，就是没法对命令行的字符使用粘帖拷贝，为了解决这个问题可以将VMware Workstation最小化到系统托盘,在后台运行虚拟机，然后使用Putty作为前台的命令行界面，利用SSH登录到ubuntu。
有关如何利用putty以及SSH，可以参考

  • ubuntu下安装配置SSH，Putty。
  • putty使用技巧(珍藏版)

其中有个技巧很好，可以解决ssh登录慢的问题。其实是sshd的问题，跟putty没关系。方法如下：

首先修改sshd配置文件

sudo vi /etc/ssh/sshd_config

增加一行:

UseDNS no

然后重启sshd服务

sudo service ssh restart

锦上添花之二，最小化VMWare到系统托盘

通常情况下最小化VMware Workstation后偌大一个长条占据了有限的任务栏(Taskbar)空间着实令人不爽。为解决这个问题，可以通过VMware的界面进行操作。以VMWare Workstation版本是10.0.5 build-2443746为例。打开菜单：”编辑/Edit”->”首选项/Preferences”，在弹出的对话框中选中”工作区/Workspace”,把”Workstation关闭后保持虚拟机运行/Run powered on virtual machine in background after close”选中。

这样以后当关闭VMware Workstation 的时候，VMWare即可在后台运行，同时任务栏的程序就不见了，VMware被最小化到了右下角的系统托盘。

锦上添花之三，固定IP

缺省情况下Ubuntu采用动态IP地址，所以用Putty登录时经常会发现有时候系统重启后Ubuntu的IP地址就变了，有点麻烦不是吗，所以就把它改成静态IP地址就好了。
修改文件：/etc/network/interfaces

$ sudo vi /etc/network/interfaces

在文件中对你的网络接口，我这里是eth0做如下配置：
如果有语句形如iface eth0 inet dhcp，坚决注释掉。
添加如下语句：

iface eth0 inet static
address 192.168.74.210
netmask 255.255.255.0
gateway 192.168.74.2

第一句是设置eth0为静态配置，下面三句分别是配置ip地址，网络掩码和网关IP地址，请按照自己的实际情况修改即可。
改好后保存退出，重启系统即可生效，这样你的Ubuntu的IP地址就不会变了。

除了该IP地址，DNS服务器的IP地址可能也需要手工改一下，因为现在我们把DHCP给禁止了。注意我这里针对的是Ubuntu 12.04LTS，如果仅仅修改’/etc/resolv.conf’，重启后修改就会丢失，解决的方法如下，参考Ubuntu 12.04/12.10 中自定义DNS服务器设置 ，总结如下：
首先我们需要创建一个文件/etc/resolvconf/resolv.conf.d/tail

$ sudo touch /etc/resolvconf/resolv.conf.d/tail

然后我们在这个文件里写入自己要添加的DNS服务器，格式与以前的/etc/resolv.conf文件一致，譬如：

nameserver 114.114.114.114
nameserver 8.8.8.8

接下来我们重启下resolvconf程序，让配置生效：

$ sudo /etc/init.d/resolvconf restart

再去看看/etc/resolv.conf文件，自己添加的DNS服务器果然乖乖的写进去了！至此问题完美解决！

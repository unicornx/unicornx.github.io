工作环境：

Windows XP/Windows 7

预置条件：

Windows上已经安装git

已经在github上创建好pages

步骤

step 1: install node.js

download latest installation package from nidejs homepage

step 2: open git-bash

为了加速访问，很多国外的源由于被墙的原因，速度很慢。幸好有一个国内的镜像，输入如下指令设置国内的源：

$ npm config set registry http://registry.cnpmjs.org

20150724更新，今天在一台干净的机器上重新做了一遍，做到这步发现不同。

首先是设置了http://registry.cnpmjs.org的源后老是下不来，所以上http://cnpmjs.org/去看了一下，不知道是不是更新了还是咋的，反正我根据网页的提示”中国用户请访问国内镜像站点”，点击后就来到了https://npm.taobao.org/。按照提示我更新了源如下：

$ npm config set registry https://registry.npm.taobao.org/

然后执行步骤照常，唯一的区别是原来的第4步不需要做了，其实我觉得也是，干嘛要自己设置环境变量呢。

20151129更新，今天在一台干净的Windows7机器上重新做了一遍，也发现只要设置taobao的源即可，同时第4步不需要做。

关闭bash：close git-bash
参考：Npm包下载不成功及Github打不开解决方案

step 3: install hexo

open git-bash, 使新的源设置生效

$ npm install -g hexo

close git-bash

step 4: set env vars（这一步最新的尝试发现可以不做）

20150724更新，用了最新的源后这步不做也可以。
add hexo 执行路径 to env PATH
C:\Program Files\nodejs\node_global\node_modules\hexo\bin
add new env var NODE_PATH=C:\Program Files\nodejs\node_global\node_modules

step 5: verify hexo works

open git-bash 使新的env vars生效

$ hexo version

20150724更新，用了最新的源后下面这步不做也可以，似乎也没有啥问题。

$ node
> require('hexo') // this will see the hexo vesion is 3.1.1

step 6: init hexo

$ cd myblog //进入你blog工作目录的主目录
$ hexo init
$ npm install
$ npm install hexo-deployer-git --save

参考install deployer for v3, in current folder

step 7: configure site

  • title: # 修改为blog站点的标题
  • subtitle: #修改为blog站点的副标题
  • description: #修改为blog站点的描述，可以被搜索引擎作为关键字搜索
  • author: # 站点作者名字，会显示在主页页脚位置
  • email: # 站点作者的电子邮件联系方式，
  • language: # 站点显示的文字编码格式，我这里使用的是zh-Hans，主要是和我使用的主题是next有关，详细参考next特性-设置语言
  • url: # 修改为本站点的URL
  • deploy:
    如果你的git仓库使用的是github，那么可以做如下配置：
    type: git
    repository: #你的github上project的URL，一般形如：https://github.com/yourusername/yourusername.github.io.git
    branch: master
    如果你的git仓库使用的是gitcafe，那么可以做如下配置：
    type: git
    repository: #你的github上project的URL，一般形如：git@gitcafe.com:yourusernamx/yourusernam.git
    branch: gitcafe-pages

step 8: compile site code

$ hexo clean 
$ hexo g

step 9: publish

$ hexo d

完成后上网站看看：http://unicornx.github.io/

后继的一些站点配置工作总结

设置主题/theme

我选择了是next，具体参考next on github

下载了该主题后，相关代码都放在”themes/next”下。
首先修改：开启紧凑模式，将主题配置文件”themes/next/_config.yml”中的

#scheme: Mist

参考next-创建分类页面

一个类别的例子：

categories: [internet]

多个类别：

categories: [internet, tools]

修改你编辑网站文件的模版

在你blog工作目录的scaffolds下有三个文件：

  • draft.md # 草稿文章模版
  • page.md #页面模版
  • post.md # 发布文章的模版

以post.md为例，这个模版会影响如下行为：
当我们要发布文章时，需要在工作目录下运行如下命令：

hexo new [template] “文章名字”

template不写则缺省为post，命令执行完后会在”source/_posts”目录下生成同名的md文件，譬如：文章名字.md，缺省内容就是从scaffolds/post.md拷贝过来的。
我喜欢的模版样式： 

title: {{ title }} # 这里会自动插入文章名字
date: {{ date }} # 这里会自动插入new文章时的时间
categories:
tags:
--- # 其后为正文

<!--more--> # 这个可以用来做出“Read More”的效果，在这行前编辑的正文会显示在页面上，这行后编辑的正文则隐藏，点击“Read More”后会跳转到本文页面继续显示。

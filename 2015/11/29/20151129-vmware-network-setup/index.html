嵌入式开发环境中如何设置VMware的网络

发表于 2015-11-29   |   分类于 开发环境与工具   |  

文章系原创，如需转载请注明：转载自”Blog of UnicornX” (http://unicornx.github.io/)

最新更新于：2016-04-04

这两天找了点时间总结了一下VMware里配置网络参数的方法，由于工作中经常需要在Windows里用VMWAre，但网络这一块以前也没搞得太清楚，这次总结一下备忘。

首先要知道的是VMWare虚拟机网络的三种工作模式。这篇文章总结的很好了，所以我就不再赘述。

如果不考虑和开发板连接调试的问题，可以采用最简单的NAT方式。

但如果需要连接开发板的时候就要使用“桥接”模式了。这里有几种配置方式：

首先定义一下：

  • Host: 在VMWare的手册中指安装VMWare的宿主系统，我这里当然指的是台式机或者笔记本上安装的Winodws系统。
  • Client: 在VMWare的手册中指安装在WMware里的系统，我这里是指在VMWAre里安装的Ubuntu系统
  • Dev-board: 开发板
    注：这里Host和Client是在一台电脑上的，所以用方框框起来了，称之为主机。

三种针对不同场景的配置方式：

1) 如果仅考虑需要让Host系统，Client系统和开发板之间互通，可以直接在VMWare里采用“桥接”模式，同时开发板和主机之间采用直连（交叉）网线直接连接。如此这般设置后，Host，Client和Dev-board三者之间就好比是连接在同一个网段中的三台主机。这种方式只能实现Host，Client和Dev-board之间互通，如果要上internet就不行了。

2) 如果要和外部主机连接，譬如要上internet，又譬如在教室上课的时候，我在我的电脑上操作Client的Ubuntu和Dev-board通讯，同时又要让教室里的其他同学看到我的电脑的屏幕。如果要这样的话，需要引入Hub的支持，联网方式如下：

3) 如果不是在教室里，譬如在办公室或者是家里，不想带着一个Hub走来走去，而且现在开发的时候主机都是一台笔记本，笔记本一般同时具备无线网卡和有线网卡，我们可以利用无线网口上网，同时利用笔记本的以太网口和开发板用直连网线（交叉方式）直接连接进行调试，但有用户发现在使用有线网卡进行络调试的时候，往往会与无线网卡冲突。此时可以对VMWare做一些设置解决这个问题，具体方法，参考”华清远见开发环境使用说明 V12B 2.1”的5.2节”解决上网与本地网络调试冲突”。

操作过程中注意以下几点：

  • 第一步，我使用的是VMWare Workstation，所以直接到VMWARE的安装目录下运行vmnetcfg.exe。
  • VMNet0对应无线网卡(桥接模式)，VMNet2对应有线网卡(桥接模式)。
  • Client中的eth0对应无线网卡的网段，eth1对应有限网卡的网段

大致思路是为虚拟机Client设置两个网络适配器，一个是eth0，对应VMNet0，和Host的wifi接口桥接在一个网段里，另一个是eth1，对应VMNet2，和Host的有线网口桥接在一个网段里，这样我的虚拟机可以通过主机的有线网卡和开发板通讯进行调试，同时通过主机的wifi接口上internet了。

对应虚拟机client网卡配置如下：

auto lo
iface lo inet loopback

#############################################
# Bridge for WLAN
auto eth0
iface eth0 inet static
address 192.168.0.2
netmask 255.255.255.0
gateway 192.168.0.1
dns-nameservers 114.114.114.114

#############################################
# Bridge for Fixed-Line
auto eth1
iface eth1 inet static
address 192.168.5.21
netmask 255.255.255.0
gateway 192.168.5.1

设置时还要注意一个问题，就是发现在Windows XP里默认有线网卡优先权高，即有线网卡和无线网卡同时激活则上网优先走有线网卡。但是我这里却需要采用无线网卡上网。这个问题一开始我在Windows7里没注意，也不记得是不是曾经在Windows7里设置过，总之如果需要优先改为无线上网优先，需要对host的Windows系统网络配置做相应调整如下：

  • 进入网络属性的有线网卡的连接属性，选择TCP/IP属性，点“高级….”，进入高级TCP/IP属性页面，把网关删除掉（如果有的话），把下面的“接口跃点数”改为“2”或更高。
  • 同样进入无线网卡TCP/IP设置，设置好网关，把接口跃点数改为“1”。
  • 如果无线网卡的“高级TCP/IP属性”中的“常规”页面里有“在远程网络上使用默认网关”的选择的话，把它钩上。

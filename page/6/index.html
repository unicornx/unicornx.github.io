U-boot和内核之间传递复杂参数的TAG方式

发表于 2015-12-26   |   分类于 Linux内核   |  

文章系原创，如需转载请注明：转载自”Blog of UnicornX” (http://unicornx.github.io/)

参考文章：

  • U-boot如何向kernel传递参数 + kernel如何读取参数
  • Linux内核参数传递Tag(init_tags)

在2.4（具体哪个版本记不清了）以后的Linux内核中引入了一种新的向内核传递参数的方法tag标记。以下代码参考的是Linux内核v2.6.32.6。参考的主要文件集中在arch/arm/kernel/setup.c，该文件中有下面这么一段注释，基本说清楚了当时引入TAG的思想。

/*
 *  Tag parsing.
 *
 * This is the new way of passing data to the kernel at boot time.  Rather
 * than passing a fixed inflexible structure to the kernel, we pass a list
 * of variable-sized tags to the kernel.  The first tag must be a ATAG_CORE
 * tag for the list to be recognised (to distinguish the tagged list from
 * a param_struct).  The list is terminated with a zero-length tag (this tag
 * is not parsed in any way).
 */

用于传递复杂参数的TAG内存区

在ARM架构上，bootloader向Linux内核传递参数利用了R0，R1和R2三个寄存器，并采用如下约定：

寄存器 约定
R0 暂时不用，缺省放0
R1 机器号，标识计算机系统的型号，内核支持的所有使用ARM处理器的设备ID号定义在arch/arm/tools/mach-types文件中，编译内核过程中会被转换为一个头文件include/asm-arm/mach-types.h供其他文件包含使用。
R2 R2寄存器传递的是一个地址，也就是指针的概念，这个指针指向一个TAG区域．UBOOT和Linux内核之间正是通过这个扩展了的TAG区域来进行复杂参数的传递，如 command line，文件系统信息等等，用户也可以扩展这个TAG来进行更多参数的传递。

Linux 2.6.32.2内核的Documentation/kbuild/makefiles.txt

发表于 2015-12-21   |   分类于 Linux内核   |  

文章系原创，如需转载请注明：转载自”Blog of UnicornX” (http://unicornx.github.io/)

最新更新于：2015-12-21

方便自己阅读而已。

有关内核的Make工作机制，总结如下，参考了《嵌入式Linux应用开发完全手册》Page303：

  • (1) 配置文件.config中定义了一系列的变量形如CONFIG_FOO，Makefile将结合它们来决定哪些文件被编译进常驻内存的内核、哪些文件被编译成模块。
  • (2) 顶层Makfile决定哪些一级子目录参与编译、arch/$(ARCH)/Makefile决定arch/$(ARCH)目录下哪些文件参与编译。置于具体编译为常驻内核镜像还是模块，参考(1)中的配置。
  • (3) 各个一级子目录下的Makefile决定所在目录下哪些文件参与编译，以及继续递归进入哪些下层子目录进行编译。置于具体编译为常驻内核镜像还是模块，参考(1)中的配置。
  • (4) 顶层Makefile和arch/$(ARCH)/Makefile设置了可以影响所有文件的编译、链接选项，譬如：CFLAGS,AFLAGS,LDFLAGS,ARFLAGS。
  • (5) 各级子目录下的Makefile文件中可以设置能够影响当前目录下所有文件的编译和链接选项，譬如：EXTRA_CFLAGS,EXTRA_AFLAGS,EXTRA_LDFLAGS,EXTRA_ARFLAGS；还可以设置只影响某个文件的编译选项：CFLAGS_$@,AFLAGS_$@。
  • (6) 顶层Makefile按照一定顺序组织文件，根据链接脚本arch/$(ARCH)/kernel/vmlinux.lds生成最终的内核镜像文件vmlinux。

MMU学习笔记(基于ARM920T)

发表于 2015-12-16   |   分类于 ARM裸机接口   |  

文章系原创，如需转载请注明：转载自”Blog of UnicornX” (http://unicornx.github.io/)

最新更新于：2016-03-25

1. 前言

本文的主要目的是结合S3C2440芯片使用的ARM920T处理器来学习虚拟存储的概念，在虚拟存储器实现的过程中除了涉及MMU外，还涉及TLB，Cache的概念，在这里也一并描述。

首先提一下我的这篇文章的主要参考，首先当然是技术手册：

  • ARM920T Technical Reference Manual (Rev 1)
  • S3C2440A User Manual (Revision 1)

网上的文章包括：

  • S3C2410 MMU（存储器管理单元）详述 这篇文章对MMU的有关物理地址到虚拟地址的映射的工作原理讲得非常好，值得一读, 理解了MMU实现地址映射的工作原理后再结合ARM920T手册中的实际芯片的工作机制，特别是section机制再想想应该就明白了。

  • （六）、MMU的使用 这篇文章对一些基本概念讲的不错，特别地有一张总结的表格很好，但对地址映射问题这篇文章讲得并不是很明白。

  • Advanced Compilers and Architectures - ARM MMU Overview 这篇文章应该是从一个更高的层次在介绍内存管理技术，而不仅仅是MMU，其中提到了一些概念值得以后深入思考。譬如：

    • 内存管理的两种技术：Two Major Technologies，包括Segmentation和Paging。本文讨论的实现方式应该就是对应的Paging方式。
    • 内核里VM的处理方式，参考Page Allocation。

    • ARMv7架构的内存管理方式：

      ARMv7 comes in 3 variants:
• ARMv7-A with MMU (paging).
• ARMv7-R for hard realtime applications with MPU (segmentation).
• ARMv7-M micro-controller version, no memory protection.
